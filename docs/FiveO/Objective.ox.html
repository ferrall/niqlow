 Objective.ox

Derive MyObjective from an Objective class.  ⇩ 

A user defines a non-linear objective as a class derived from Objective. Currently the main types are:

  • UnConstrained: an objective to maximize without imposing equality and inequality constraints on paramers. In turn, unconstrained objectives should be derived from one of three sub-classes:
    • A BlackBox: the objective is a BlackBox objective.
    • Separable: the objective is the Separable sum of K blackbox objectives. Some parameters can be common across the objectives.
    • Mixture the objective is a Mixture: a separable mixture of K problems over G different environments. A separable objective is a special case where the G=1 and weights are fixed.
  • Constrained: an objective to maximize subject to equality and inequality constraints using sequential quadratic programming (SQP).
  • System: A square system of non-linear equations for which roots are to be found. See Non-Linear System for details.

The objective \((\psi)\) is provided by the user as MyObjective::vfunc().
The v indicates that it can return the vector of values that, if summed up, is the value of the objective at the current parameter values.

Parameters to choose to optimize the objective are represented by an object derived from the Parameter class.
Different parameters classes are defined to constrain individual parameters to open regions.
Related parameters are handled in a ParameterBlock. See Parameters for details.

Parameters are added to the objective using Parameters().

The work of optimization of the objective (or solution to the system) is done using a method of the Algorithm class.
When an algorithm object is created the objective is sent as a argument and is assigned as a constant member. So during execution an algorithm object is associated with a single objective, but multiple algorithms can be applied to the same objective. This allows for a comparison of algorithms and/or applying a sequence of algorithms in order to carry out the optimization.

Blackbox Objective

A blackbox is a function f:ℜN→ℜ.

Separable Objective

A separable objective is the sum of K BlackBox objectives. A separable objective is a function and a parameter matrix of the form

$$\eqalign{ \psi \quad &=\quad \left(\psi_0 \psi_1 \dots \psi_{K^-} \right)\cr f(\psi) \quad &=\quad \sum_{k=0}^{K^-1} \ \sum_{} f_k\left(\psi_k)\right) \cr }$$

Each sub-vector ψk is of the same length, (ψ.k).N = M.. The overall parameter ψ is then a M × K matrix, the horizontal concatenation of the the sub-vectors. Obviously, for K=1 a separable objective is equivalent to a BlackBox.

The user only needs to code the sub-objective as a function of the sub-parameter vector and a Discrete quantity, K. That is, the user codes vfunc() to equal fk(ψk). The current value of the index k is stored as Kvar.v.

To create a separable objective, call new Separable(L,Kvar), where Kvar can be either 

Kvar = positive integer
 or
Kvar = objective of class Discrete
If a positive integer is sent then FiveO sets this.Kvar = new Discrete("K",Kvar);. Otherwise, the user sent discrete variable is used directly.

Algorithms do not know the difference between separable objectives and blackboxes. Instead, FiveO keeps track internally of the matrix structure of the parameter vector.

Ordinarily a single parameter of a separable objective spawns K parameter values. Parameters(x) creates a single parameter (or block), but the value of that parameter will be allowed to vary across k. Each value of a parameter is under the control of algorithms. Any dependency between parameters within a sub-vector will be respected k by k.

A parameter can be added to the objective with Common(). Now the parameter has only one value which will be the same across k. If no common parameters (and no Determined parameters) are added to a separable objective then the length of the (overall) parameter vector seen by an algorithm is N = KM. C≤M denotes the number of common parameters that were added to the objective. In general, the overall parameter vector has length N = C+K×(M-C).

Parameters are always written out to files with duplicates, so it will appear that N×K parameters exist, but common parameters will always be equal across k. This means that parameters can be changed from common to specific and vice versa without modifying files created by Save().

The overall parameter vector concatenates the sub-vectors:< 

ψ ≡ 〈 ψ0 ψ2 … ψK‾ 〉

Coding a Separable Objective.

The user codes the component function, fk(ψk). as the vfunc() method. vfunc knows which sub-problem is active through the index Separable::k. From the coding point of view vfunc, there is one objective which might be written fk(ψk) ≡ g(ψ,k) rather than K different computer-coded functions. Five-O sets k and places the correct parameter values in ψ before calling vfunc().

Separable::Parameters() creates a type-specific parameter that varies across k. So Parameters(x) creates K different parameters to be optimized over, one for each sub objective fk(ψk). 

Separable::Parameters(x)  →  ψ = 〈 (x0 …) (x1 …) … (xK‾ …) 〉
In many applications, some parameters are common across sub-objectives. A common parameter is added using Common(). So Common(y) creates only one parameter to optimize and its value will be the same for each k as vfunc() is called. 
Separable::Common(x)  →  ψ = 〈 (x …) (x …) … (x …) 〉

If no common parameters are added to the objective then the length of the (overall) parameter vector is N = KM. C denotes the number of common parameters. The overall parameter vector has length N = C+K×(N-C). Parameters are always written out with duplicates, so it will appear that N×K parameters exist. This means that parameters can be changed from common to specific and vice versa without modifying files created by Save().

Mixture: Separable objective with linearly weighted sub-objectives.

A Mixture is the objective from a finite mixture model of unobserved heterogeneity, in which the separate functions involve two stages and weighting parameters. A Mixture generalizes a separable objective in two ways, so it is a class derived from Separable. First, a mixture includes a second discrete dimension, Dvar. The current value, Dvar.d, takes on values 0 … D‾ and indexes different environments that the parameters enter. Now the sub objective depends on k and d. However, parameter vectors continue to only depend on k. Thus, in a mixture model sub-objectives take the form ƒk d(ψk). There are KD different combinations. Second, a mixture has a separate D×K matrix of weighting parameters, denoted λ.

The mixture objective :
ƒ = ∑d=0D‾  ∑k=0K‾  λ[d][k] ƒk d(ψk)


Typically (but not necessarily), the weights are each a unit Simplex for each d:
  0 ≤ λ[d][k] ≤ 1
  ∑k=0K‾ λ[d][k] = 1.

A separable objective is the special case of a mixture with D=K and λ = IK. That is, each type is specific to an environment.

To create a mixture objective, call new Mixture(L,Dvar,Kvar,...) , where Kvar is the same as for separable objectives and Dvar can be the same: 

Dvar = positive integer
 or
Dvar = objective of class Discrete
If a positive integer is sent then FiveO sets this.Dvar = new Discrete("D",Dvar);. Otherwise, the user sent discrete variable is used directly.

The variable argument are optional initializing values for λ 
Let va = va_arglist();
No optional argument (va = {}) means
	lambda[d] = new Simplex("L"+sprint("%2u",d),K),  for d = 0, … D‾
A single argument (va = array[1]), then it must be a D×K matrix
	lambda[d] = new Simplex("L"+sprint("%2u",d),va[0][d][]),  for d = 0, … D‾
D arguments (va = array[D]), then each must be a ParameterBlock of size K and
	lambda[d] = va[d]    for d = 0, … D‾

The final option allows the user to use weights that do not sum to 1 for each value of d, because the parameter block need not be a Simplex parameter block. It is inefficient to compute ƒk d if its weight is zero. The user can send a matrix of 0s and 1s to IncludedDK(), which indicates which type-environment pairs should be ignored. The default matrix consists of 0s, meaning that no combinations are ignored. Algorithms do not know any difference between a mixture, separable or blackbox objective. Algorithms will work with a concatenation of λ and ψ. Common parameters operate the same way in a mixture as in a separable objective.

If no common parameters (and no Determined parameters) are added to a separable objective, and the weights are D K-Simplexes, then the length of the (overall) parameter vector seen by an algorithm is N = D(K-1) + KM. In general, the overall parameter vector has length N = DK + C+DK×(M-C).

Objective :: ToggleParameterConstraint ( )
Toggle the value of DoNotConstrain. If DoNotConstrain then all parameters except Determined parameters are free and unscaled. This can improve convergence near the top of the objective. It also means that the Hessian values are in terms of the economic parameters not the free-to-vary optimization parameters.

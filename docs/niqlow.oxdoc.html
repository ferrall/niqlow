DESIGNSOLVEESTIMATE
Design a DP model using DDP. Solve the model with a solution Method and create Data outcomes from the solution. Estimate model parameters using FiveO Objective and solution Algorithms.
See a search model in Get Started with DDP or start from scratch with static discrete choice and go from there or follow replications of classic papers in OODP Maximize a function in Get Started with FiveO
Build your DP in niqlow
  • Choose a base Bellman class for each point \(\theta\) of the state space.
  • Customize \(\theta\) by adding components to it
    • Set the Clock
    • Add Action Variables to create your action vector \(\alpha\).
    • Add State Variables to \(\theta\), from which the model's transition emerges
    • Create multiple problems by adding Group Variables
    Customize the DP problem further
    • Code Utility()
    • Make values of state variables Terminal Values
    • Restrict actions depending on \(\theta\) by coding FeasibleActions() to define \(A(\theta)\).
    • Restrict which states can be reached beyond niqlow's built-in understanding of transitions by coding Reachable()
    • Let niqlow handle the mapping between discrete counter values of variables and their actual values that enter your formulas
    • Set the point in the solution algorithm when transitions must be recomputed depending on the reliance on parameters or group variables without computing the transitions unnecessarily.

    When finished, tell niqlow to Create the spaces

      Once Spaces are created
    • Choose a solution method. Let the method efficiently handle storage and computation accounting for different clocks and convergence criteria they imply.
    • Create a Panel to simulate outcomes solved model or read in external data to compute likelihood.
    • Create a PanelPrediction to compute expected outcomes (averaged over all random elements) or read in external moments to carry out GMM estimation
    • Map components of the model outcome to the data and account for missing information.
    • Add auxiliary outcomes to the data to partially make up for missing action and state variables.
    • Nest a solution method in a data object to carry out a nested solution algorithm.
    • Track the amount of time spent solving the PD versus simulating or predicting outcomes.
      Once the map from the model to data/outcomes is complete estimate or calculate parameters of the model
    • Put parameters of the DP, such as preference parameters or prices, under the control of other algorithms by making them Parameters.
    • Use the built-in types of parameters to constrain them to open intervals in one or more dimensions. Use Parameter Blocks to create convenient vectors of related parameters.

    • Use an automatically generated econometric objective or build your own.
    • Create a system of equations to find Equilibrium prices.
    • Optimize the objective or solve the system by applying one of the built in Algorithms
    • Apply different methods, monitor and checkpoint progress
    • Use the estimated/equilibrium parameters to conduct policy experiments and hypothesis tests

    When your model becomes too big to solve on your personal computer: use the integrated CFMPI package to carry out solution and estimation in parallel on a Linux cluster. With some set up a user of niqlow can be using MPI on their program by add

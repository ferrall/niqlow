 EMaxIteration.ox

Expected Value Function Iteration..
EMaxIteration is derived from the DP class. Classes based on EmaxIteration code different specifications of the iid continuous state variables in the model, denoted ζ in niqlow.
A user would base MyModel on one of these classes.

They also code the form of Bellman's equation for the specification which integrates out call the IID state vectors, ζ, ε and η. 
Emax ≡ EV(θ,γ)  ≡  ∫ζ  ∑ε  ∑η V(ζ,εη,θ,γ)
The term EmaxIteration is used instead of EVIteration because EV is used for extreme value specification of ζ Further, Emax is often used for this calculation because V() is the max over feasible actions at the state.
The method to compute this at a point in the endogenous state space is EmaxIteration::Bellman(). Each derived class has a version of Bellman that accounts for the integration. EmaxIteration::Bellman() is virtual, so a user can provide an alternative their own specifications to compute MyModel.
Each EmaxIteration class also has a version of EmaxIteration::Vsolve(), which traverses the state space Θ computing Emax. Vsolve() updates the transitions and the utility values so that changes in parameters are accounted for during repeated solutions. Vsolve() can be told to traverse Θ a fixed number of times or it will iterate until V() converges and/or backwards in time.

There are two related aspects of the environment embodied in each EmaxIteration method.

Smoothing: The form of continuous IID choice-specific errors in the state.
In the model specification the ζ vector contains continuous IID shocks to U(). The base EmaxIteration class is equivalent to an empty ζ.
This relates primarily to how choice probabilities are smoothed, so is referred to as the smoothing method. But obviously the distribution of continuous states can be integral to the model's prediction.
Solution: The algorithm to compute EV, referred to as the solution method.
The solution method is affected by the size of the state space and the type of choices. Smoothing and solving are not fully independent, but in some cases different algorithms can be applied to the same smoothing option.

Another way to think of the distinction: smoothing primarily relates to integration over choice-specific dimensions of the state space, conditional on the discrete state. Solution relates to integration over the rest of the state space.

With two smoothing methods, A and B, and two solution methods, I and II, one might want to create a problem like this: 
class MyClass : A I
However, in Ox a derived class has only one parent: it cannot combine features of two separate classes. Therefore, four separate classes have to be defined, AI, AII, BI, and BII.

Hybrid Solution Methods
Some methods for solving DDP integrate data to be modeled into the solution of the program. And others rely not on iteration but on non-linear optimization and root-finding. In niqlow these methods combine DDP and FiveO. The are categorized as Hybrid Methods and discussed separately.

 EVExAnte : ExtremeValue : EMaxIteration : DP

Additve extreme value errors enter U().
Specification 
U() = Utility(α,η,ε,θ,γ) + ζ
ζ.N = (θ.A).D          IID error for each feasible α
F(zi) = exp{ -exp{-x/ρ} }
Computations.
Computations occur at each θ ∈ Θ. θ is suppressed in the expressions until needed
Computations occur for each value of γ ∈ Γ, also suppressed until needed
Bellman Equation Iteration.
v(α;ε,η) = exp{  ρ( U + δ∑ θ′ Ρ(θ′;α,η,θ) EV(θ′) ) }
V(ε,η) = log(∑α v(α;ε,η))
EV = ∑ε,η [ V(ε,η)*f(ε)f(η)/ρ ]
Choice Probabilities
Once EV() has converged 
Ρ*(α;η) = ∑ ε [ v(α;ε,η)/V(ε,η) ] f(ε)
Storage
Ρ* is stored for each γ
EV computed above is accumulated across γ 
EV = ∑ γ EVγ f(γ)

 KWApproximation : EVExPost : ExtremeValue : EMaxIteration : DP

Approximate EV from subsample of Θ using Keane-Wolpin (1994).

KWApproximation does not allow any states in η, so it is dropped from the notation below.

Currently it also allows NormalAging only as the clock. 

Emax(θ) ≡  ∑ ε Ρε V(ε,θ)
which is the expected value of the maximum 
V(ε,θ) = max α∈θ.A  v(α;ε,θ)
When ε and θ both include many states, visiting every value of ε for every value of θ can be expensive.

KWApproximation computes Emax for a randomly selected subset of states at a given t,

ΘKW ⊂ Θ.
Actually, KW operates period-by-period, so technically it involves Θ(t), the subset of Θ with states at time t.

It interpolates Emax with a function (such as a linear regression) of the values of v(α;ε,θ) at a single exogenous vector, ε, including the non-linear transformation: 

maxE(θ) ≡ max α∈&theta.A  v(α;ε,θ)
if ε = E(ε) then this is indeed the max at the expected exogenous shock.

Then, for points θ ∉ ΘKW(t) it extrapolates the value based on only maxE(θ) and v(α;ε,θ).

The amount of computations saved is can be approximated easily. Let ε.D, Θ.D and ΘKW.D denote the cardinality of the sets (following the notations used elsewhere). Then the proportion of total max() operations avoided is 

(ε.D-1)(Θ.D-ΘKW.D) / Θ.D

KW (1994) report that the extrapolation is sufficiently accurate in their model that the simulation bias in parmeter estimates based on it is quite small.

The built-in components of KWApproximation use the KW (1994) "linear and square root" regression specification. However, these components are virtual and can be replaced by specifications or interpolating functions of the user's choice.

Since KWApproximation is derived from EVExPost the choice probabilities can be smoothed ex post.

The user must send a Clock block to Initialize() and not call DP::SetClock directly. This is because some elements of the environment have to be set before the clock is to account for subsampling of the state space.

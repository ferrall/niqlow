 Bellman.ox

Bellman's Equation and Expected Value Function Iteration. Skip down to documentation of items defined in Bellman.ox

Bellman is derived from the DP class. Classes based on Bellman capture different specifications of the model, especially in terms of the iid continuous state vector, denoted ζ.

A user would base MyModel on one of these classes (or a class ultimately derived from them).

The continuous state ζ determines the form of Bellman's for choice-specific values, which integrates out the IID state vectors, ζ, ε and η.

Solution methods are coded separately from Bellman. They are derived from the Method class and described in Methods. Some methods may only operate if the user's model is derived from a compatible class of Bellman or has other required characteristics.

Classes of type Bellman code the form of Bellman's equation for the specification: 
V(θ′) ≡  ∑η  ∑ε  ∫ζ [ V(ζ,ε,η,θ′) ] f(ζ)dζ Ρε(ε)Ρη(η)


EV(θ′) = Eα,η,θV(θ′)  = ∑θ′ V(θ′) Ρ(θ′;α,η,θ)


v(α,ζ,ε,η,θ) ≡ U(α,ε,η,θ,γ) + ζ.α + δ EV(θ′)


V(ζ,ε,η,θ) = maxα∈Α(θ)  v(α,ζ,εη,θ)
The integration over ζ is carried out by the virtual thetaEMax() or its replacement. .
The default method, thetaEMax(), assumes that ζ takes on one value, 0, with density f(0)=1. (Another way to think of it: ζ is an empty vector.) Thus, the default does no integration.
Choice probabilities are stored conditional on η and θ. This requires summing over ε and integrating over ζ Another virtual method, Smooth(), carries out the smoothing of choice probabilities, but only on a final iteration (after convergence in an infinite horizon environment).
In the default choice probabilities are not smooth in model parameters. That is, conditional choice probabilities are either 0 or 1 based on. The derived class ExPostSmoothing allows for ex-post smoothing of choice probabilities without the presence of ζ

Required and Optional User-Contributed Elements of MyModel and MyCode
MyModel must supply a replacement for Utility(), which simply returns 0.
MyModel can provide a replacement for the virtual FeasibleActions() method to make the feasible choice set to vary with the endogenous state θ.
MyModel must include a static method/function that returns a new instance of MyModel for any reachable state and 0 otherwise. MyCode must pass this method to DPparent::CreateSpaces().
MyModel can assignPreUpdate to a static method/function that will be called by UpdateVariables, which is called by Run(). This allows the user to update values of parameters before DDP recomputes transition probabilities and utility of actions.
MyModel can supply replacements for thetaEMax(), ActVal(), Smooth(). Many of the derived classes of Bellman already specialize these operations.
MyModel can add AuxiliaryVariables and a ZetaRealization for simulating outcomes and accounting for partial observability of the state.
MyCode must sandwich the commands that add actions and states to the model between calls to DPparent::Initialize(…) and DPparent::CreateSpaces(…). MyModel can supply their own versions of these two methods, by then they must call the parent versions. If MyModel does not have its own versions, then the prefix DPparent:: is not needed.

Hybrid Solution Methods
Some methods for solving DDP rely not on Bellman iteration but on non-linear optimization and root-finding. These methods combine DDP and FiveO. They are categorized as Hybrid Methods and discussed separately.

 ExPostSmoothing : Bellman : DP

Choice probabilities are smoothed ex post.

Utility() has no continuous error terms. After V() is computed, choice probabilities are smoothed ex post as follows: 

U() = Utility(α,η,ε,θ,γ)
v*(α) = exp[ρ(v(α&epsilon,ηθ)-V(&epsilon,ηθ) )]	/ ρ
Ρ*(α;ε,ηθ) = v*(α) / ∑α'∈A(θ) v*(α')

 OneDimensionalChoice : Bellman : DP

One-dimensional action models with user defined distribution of ζ.

Allows for solving the model by finding cutoffs (reservation values) in the continuous error. This solution works when there is a single action variable and there are no exogenous or exogenous states. Solving for cut-offs using non-linear systems is a Hybrid method.

Fully specified, utility is 
α = (a)
ζ=(z)
ε = ()
η = ()


U(a;;ζ,θ) = U()+E[z|a,θ]
The model must exhibit a reservation property in z.

Vsolve() computes z0 … zα.N‾ which are cutoff or reservation values for the values of z. The optimal value of a, denoted a*, is 

 a* = a  iff za-1 < ω ≤ za.
z-1 ≡ -∞
za.N ≡ +∞

 thetaEMax

virtual Bellman :: thetaEMax ( )
Default Emax operator at θ. This is a virtual method. Derived classes provide a replacement.
Computes 
v(α,ε,η,θ) = U(α,…) + δ∑θ' Ρ(θ';α,η,γ)EV(θ')
EV(θ) = ∑η,ε Ρ(η)Ρ(ε)maxα∈A(θ) v(α,...)
If θ ∈ Θ, then v(α,…) = U(α,…). Computes and EV(θ).
If setPstar then Ρ*(α) is computed using virtual Smooth()
Comments:
Derived DP problems replace this routine to account for ζ or alternatives to Bellman iteration.

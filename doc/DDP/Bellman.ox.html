 Bellman.ox

Specify Utility, Reachable States and other aspects of the state space θ.  ⇩ 

  1. Overview

    2. Bellman is derived from the DP class. Classes based on Bellman capture different specifications of the model, especially in terms of the iid continuous state vector, denoted ζ.

    A user bases (derives) MyModel on one of these classes (or a class ultimately derived from them).
    Example 
    #import DDP

    
class MyModel : ExPostSmoothing {
    // declare data members
            Utility(A);
            Reachable();
    
    // Optional methods to replace built-in versions
           FeasibleActions(A);
    static PreUpdate();
    
    
    // Optional methods to accommodate new solution methods (advanced)
           thetaEMax();
           ActVal();
           Smooth();
    
    }

    The continuous state ζ determines the form of Bellman's for choice-specific values, which integrates out the IID state vectors, ζ, ε and η.

    Bellman equation solution methods are coded separately from Bellman. They are derived from the Method class and described in Methods.
    Some methods may only operate if the user's model is derived from a compatible class of Bellman or has other required characteristics.

    Classes of type Bellman code the form of Bellman's equation for the specification:
    \(V(\theta^{\,\prime)}\quad \equiv \quad \sum_{\eta}\ \sum_{\epsilon}\left[ \int_{\zeta} V\left(\eta,\epsilon,\eta,\theta^{\,\prime}\right)f(\eta)d\eta \right] P_\epsilon(\epsilon)P_\eta(\eta)\)

    \(EV(\theta^{\,\prime}) = E_{\alpha,\eta,\theta} \left[V(\theta^{\,\prime})\right] = \sum_{\theta^{\,\prime}} \left [ V\left(\theta^{\,\prime}\right)P\left(\theta^{\,\prime}\ ;\ \alpha,\eta,\theta\right)\right]\)

    \(v(\alpha;\zeta,\epsilon,\eta,\theta) \equiv U(\alpha,\epsilon,\eta,\theta,\gamma) + \zeta_\alpha + \delta EV(\theta^{\,\prime}).\)

    \(V(\zeta,\epsilon,\eta,\theta)\quad=\quad\max_{\alpha\in A(\theta)}\ v(\alpha;\zeta,\epsilon,\eta,\theta).\) 

    V(θ′) ≡  ∑η  ∑ε  ∫ζ [ V(ζ,ε,η,θ′) ] f(ζ)dζ Ρε(ε)Ρη(η)

    
EV(θ′) = Eα,η,θV(θ′)  = ∑θ′ V(θ′) Ρ(θ′;α,η,θ)

    
v(α,ζ,ε,η,θ) ≡ U(α,ε,η,θ,γ) + ζ.α + δ EV(θ′)

    
V(ζ,ε,η,θ) = maxα∈Α(θ)  v(α,ζ,εη,θ)
    The integration over ζ is carried out by the virtual thetaEMax() or its replacement. .
    The default method, thetaEMax(), assumes that ζ takes on one value, 0, with density f(0)=1. (Another way to think of it: ζ is an empty vector.) Thus, the default does no integration.
    Choice probabilities are stored conditional on η and θ. This requires summing over ε and integrating over ζ Another virtual method, Smooth(), carries out the smoothing of choice probabilities, but only on a final iteration (after convergence in an infinite horizon environment).
    In the default choice probabilities are not smooth in model parameters. That is, conditional choice probabilities are either 0 or 1 based on. The derived class ExPostSmoothing allows for ex-post smoothing of choice probabilities without the presence of ζ

  2. User-Contributed Elements of MyModel and MyCode

    1. Utility()
      2. MyModel must supply a replacement for Utility(), which simply returns 0.
      Since utility depends on the current state, the method must be automatic (not static).
      #import DDP

      
struct MyModel : ExPostSmoothing {
    // define data members and methods
    Utility(A);
    }
MyModel::Utility(A) {
    return  {a vector of utility values given the feasible actions A }
    }
    2. Reachable state creator
      3. MyModel can optionally provide replacement for the virtual Reachable() method to return an indicator for whether the current state is reachable from initial conditions or not.
      Example 1. All states are reachable 
      
The built-in version of Reachable returns TRUE, so if all states are reachable the user does not need to provide a replacement.
      

      Example 2.  States where state variables x and y greater than 5 are not reachable

      MyModel::Reachable() {
    return ! (CV(x)+CV(y)> 5);
    }
      

    3. Restricted Feasible Action spaces / matrices 
      4. 

      MyModel can optionally provide a replacement for the virtual FeasibleActions() method to make the feasible choice set to vary with the endogenous state θ.
      

      Example.  Only action vectors with d less than or equal to the value of state variable x are feasible.

      MyModel::FeasibleActions() { return CV(d) <= CV(x); }
      

      

    4. Hooks and Update Time
      5. 

      MyModel can use Add() to have a static method/function called at different points in solution methods.
      

      MyModel can also use SetUpdateTime() to set when solution methods should update transition probabilities and utility of actions.  
      

      This allows transitions and utility to depend on fixed and random effect variables, but if they do not wasted computations can be avoided by updating higher up in the process.
      

      

    5. Value-related routines
      6. 

      MyModel can supply replacements for thetaEMax(), ActVal(), Smooth().  Many of the derived classes of Bellman already specialize these operations.
      

      

    6. Auxiliary Variables and ZetaRealization
      7. 

      MyModel can add AuxiliaryValues and a ZetaRealization for simulating outcomes and accounting for partial observability of the state.
      

      MyCode must sandwich the commands that add actions and states to the model between calls to DPparent::Initialize(…) and DPparent::CreateSpaces(…).  
      

      MyModel can supply its own version of these two methods, but then they must call the parent versions.  If MyModel does not have its own versions, then the prefix DPparent:: is not needed because
a reference to Initialize() will refer to the parent's version.

 OneDimensionalChoice : ExPostSmoothing : Bellman : DP


One-dimensional action models with user defined distribution of ζ.



Allows for solving the model by finding cutoffs (reservation values) in a continuous error using the Hybrid ReservationValues method.




The reservation value solution works when


    

  • There is a single action variable and
    • 

  • There are no exogenous (ε) or semi-exogenous (η) states added to the model.  State variables that would be eligible
for inclusion in those vectors need to be placed in θ.
    • 

  • The model must exhibit a reservation property in z (i.e. a single-crossing property and if d.N>2 monotonicity in the crossing points.
    • 

    

  • Formally,
    • 

    α = (d)
ζ = (z)
ε = ()
η = ()

    
U(d;ζ,θ) = U(d;z,θ)

    

    








The restrictions above do not apply if other solution methods are applied to a OneDimensionalChoice.




The user provides methods that return:


    

  • Uz(z): the utility matrix at a given vector of cut-offs z. Uz(z) should return a d.N × d.N-1 matrix equal to the
utility of each value of d=i at ζ=zj.  In the case of a binary choice there is just one cut-off and Uz(z) returns a column vector of
the utilities of the two choices at z  Internally the difference between adjacent values of d is computed from this matrix.
    • 

    

  • EUtility(): an array of size d.N that returns the expected utlity of d=j for values of z in the interval (z*j-1,z*j)
and the corresponding probabilities Ρ[z &in (z*j-1,z*j) ].  EUtility() gets
z*star from the data member zstar.
    •

 FeasibleActions


virtual Bellman :: FeasibleActions ( )


Default θ.A: all actions are feasible at all states, except for terminal states.


This is a virtual method.  MyModel should provide its own to replace it if some actions are infeasible at some states.


Returns:
Mx1 indicator column vector
 1=row is feasible at current state
 0 otherwise.



Example:
Suppose MyModel has a binary action d for which d=1
is feasible only if t < 10.  Otherwise, only actions with d=0
are feasible.  The following will impose that restriction on feasible actions:

MyModel::FeasibleActions() {
    return  CV(d)==0 || I::t < 10;
}





Comments:
default is to return a vector  of ones, making all actions feasible at all states.
This is not called at unreachable or terminal states.



See also:
Alpha, Actions, ActionVariable

 thetaEMax


virtual Bellman :: thetaEMax ( )


Default Emax operator at θ.
This is a virtual method.  Derived classes provide a replacement.

Computes

v(α,ε,η,θ) = U(α,…) + δ∑θ' Ρ(θ';α,η,γ)EV(θ')
EV(θ) = ∑η,ε Ρ(η)Ρ(ε)maxα∈A(θ) v(α,...)


If θ ∈ Θ, then v(α,…) = U(α,…).
Computes and EV(θ).




If setPstar then Ρ*(α) is computed using virtual Smooth()


Comments:
Derived DP problems replace this routine to account for ζ or alternatives to Bellman iteration.

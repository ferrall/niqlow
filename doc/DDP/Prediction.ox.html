 Prediction.ox

The distribution of DP outcomes, including moments and GMM.

Skip down to documentation of items defined in Prediction.ox

    1. Prediction
      2. The Outcome class is designed to work with realized results from a DP model. That is, at one point in time there is a single realization. For this reason, simulation, individual-level data and likelihood functions are related to Outcomes.

      On the other hand, the Prediction class is designed to handle the distribution of outcomes at a given time. It is designed to handle integrating out uncertainty. For this reason, prediction and generalized method of moments (GMM) are related to Predictions.

      The base class Prediction tracks a list of possible state indices and their associated probabilities of occurring. It also tracks choice probabilities.

      Starting from some initial conditions a PathPrediction is a list of predictions, each providing the distribution across states and choices at that point in time. A PanelPrediction is a set of PathPredictions across fixed effect groups. Data can be read in to match up with predictions using the EmpiricalMoments class derived from PanelPrediction.

       
       
      Prediction
       
      		 The expected outcome from a single realization.
       
       
       
      PathPrediction
       
      		 A derived Prediction that points to a sequence of predictions along a single realized DP path. As an prediction, the path itself is the first prediction along the path. Since random effects are assumed to be unobserved, the path prediction integrates over any random effect groups that are in the model. There is only one prediction for a given fixed effect group, so there is nothing like the FPanel class derived from Outcomes. Each PathPrediction is in effect a FixedPathPrediction
       
       
      		  
       
      PanelPrediction
       
      		 A list heterogenous path predictions. It is a PathPrediction which can also point to more PathPredictions. As a PathPrediction itself, the PanelPrediction is the first path in the panel.
       
       
      		    
       
      EmpiricalMoments
       
      		 A panel prediction with data-handling tools. Unlike the other derived classes, empirical moments do not point to other panel predictions.

     Histogram

    PanelPrediction :: Histogram ( printlevel , UseDist )
    Compute histogram(s) of an (array) of objects along the path.
    Parameters:
    prntlevel CV compatible print level
    Zero (default): silent
    One : formatted print each object and time
    Two: create a flat matrix of moments stored in PathPrediction::flat.
    UseDist TRUE [default]: use endogenous choice probabilities Ρ*
    FALSE : use uniform choices.

    Predict() must be called first.

    vlso, if prntlevel==Two leave in gmm the total distance between predicted and empirical moments

    Currently the objective is the square root of the squared differences.

    Returns:
    flat matrix of predicted moments
    Example:
       pd = new PathPrediction();
   pd->Tracking({capital,labour});
   pd -> Predict(10);
   pd -> Histogram(TRUE);  //print distribution

     PathPrediction

    PathPrediction :: PathPrediction ( f , method , iDist )
    Set up predicted distributions along a path.
    Parameters:
    iDist initial distribution.
    integer: start at iDist and increment until a reachable state index is found. So PathPrediction(0) [default] will start the prediction at the lowest-indexed reachable state in Θ.
    matrix: a list of states to start the prediction from
    object of Prediction class: use sind as the initial state for this prediction.

    The prediction is not made until Predict() is called.

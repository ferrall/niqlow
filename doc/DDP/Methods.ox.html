 KeaneWolpin : ValueIteration : Method : Task : DP

Approximate EV from a subsample of Θ using Keane-Wolpin (1994).

KWApproximation does not allow any states in η, so it is dropped from the notation below. 

Emax(θ) ≡  ∑ ε Ρε V(ε,θ)
which is the expected value of the maximum 
V(ε,θ) = max α∈θ.A  v(α;ε,θ)
When ε and θ both include many states, visiting every value of ε for every value of θ can be expensive.

KWApproximation computes Emax for a randomly selected subset of states at a given t,

ΘKW ⊂ Θ.
Actually, KW operates period-by-period, so technically it involves Θ(t), the subset of Θ with states at time t.

It interpolates Emax with a function (such as a linear regression) of the values of v(α;ε,θ) at a single exogenous vector, ε, including the non-linear transformation: 

maxE(θ) ≡ max α∈&theta.A  v(α;ε,θ)
if ε = E(ε) then this is indeed the max at the expected exogenous shock.

Then, for points θ ∉ ΘKW(t) it extrapolates the value based on only maxE(θ) and v(α;ε,θ).

The amount of computations saved is can be approximated easily. Let ε.D, Θ.D and ΘKW.D denote the cardinality of the sets (following the notations used elsewhere). Then the proportion of total max() operations avoided is 

(ε.D-1)(Θ.D-ΘKW.D) / Θ.D

KW (1994) report that the extrapolation is sufficiently accurate in their model that the simulation bias in parmeter estimates based on it is quite small.

The built-in components of KWApproximation use the KW (1994) "linear and square root" regression specification. However, these components are virtual and can be replaced by specifications or interpolating functions of the user's choice.

 Solve

AguirregabiriaMira :: Solve ( Fgroups , inmle )
Solve Bellman's Equation..
Parameters:
Fgroups DoAll, loop over fixed groups
non-negative integer, solve only that fixed group index
MaxTrips 0, iterate until convergence
positive integer, max number of iterations
-1 (ResetValue), reset to 0.

This method carries out Bellman's iteration on the user-defined problem. It uses the ClockType of the problem to determine whether it needs to find a fixed point or can simply work backwards in time.

It uses a FixedSolvetask stored in ftask to loop over fixed effect values.

If UpdateTime is OnlyOnce (see UpdateTimes), then transitions and variables are updated here. 

    Solve
       FixedSolve                      loop over fixed effects
            RandomSolve                loop over random effects given fixed values
                Gsolve                 solve for the point γ ∈ Γ in group space [virtual]
                    EndogUtil            initialize over endogenous states θ
                    1. Run                  loop over bellman iterations
                    2. Update         check convergence/work backwards (repeat 1.)
            `ValueIteration::PostRESolve'                          user-defined post random solve tasks

Comments:
Result stored in VV matrix for only two or three ages (iterations) stored at any one time. So this cannot be used after the solution is complete. EV stores the result for each reachable endogenous state.
Results are integrated over random effects, but results across fixed effects are overwritten.
Choice probabilities are stored in pandv by random group index

 KeaneWolpin

KeaneWolpin :: KeaneWolpin ( myKWEMax )
Initialize Keane-Wolpin Approximation method. KW Approximation computes complete "brute force" max v(α) operator on a randomly chosen subsample of points in the endogenous state space to predict (extrapolate) choice probabilities at the non-sampled states without max v(α) computations.

Semi-endogenous states, η, are not supported in this method.
All state variables must be either fully endogenous are placed in the endoenous vector γ.
Different feasible sets A(θ) are allowed, but ...
The feasible set must be the same size at each state at a give clock setting t.
A warning message about this is issued if more than one feasible set exists

The key elements of the approximation are

  1. The points to subsample at each clock setting.
    • A different proportion can be used at each t, and the approximation can be turned off completely at other values of the clock. This subsampling is done by calling SubSampleStates(profile), where profile can be a vector of sampling proportions or a constant.
    • At the subsampled points in θ all the exogenous states are iterated over to compute the full value function.
    • This value of the value is stored as the explained value for the approximation.
    • The explanatory values are also stored, in the default, the choice-specific values at the MEDIAN point in the exogenous vector ε and the max of them.
  2. The MEDIAN exogenous state to use for prediction (see above)
    • If an odd number of discrete points are used for each exogenous random variable, and if you specific a symmetric distribution of actual values then the MEDIAN point will be the 0 vector of ε and it will also equal the mean ε.
  3. The Specification of the approximation
    • KW's preferred specification is the default, but it can be replaced by the user (no help yet available on this).
    • The default is to run a linear regression in the V-v(α) vector and the square root of the vector

virtual KWEMax :: Run ( th )
Carry out Keane-Wolpin approximation at an endogenous state θ .
Parameters:
th θ There are three conditions upon entering this routine at θ
  1. θ is in the subsample of complete solutions and this is the first pass.
    2. In this case nothing needs to done further and the function returns
  2. θ is not in the subsample of complete solutions and this is not the first pass
    3. In this case the value of the state is interpolated by computing V at the median exogenous state then predicting the expected value across all exogenous states.
  3. θ is in the subsample and this is the first pass
  4. Bellman is applied to all exogenous states at this endogenous state θ
    The result is added to the KW sample

 Install

Acquire, Install and Use niqlow.
    Steps
  1. Get and install Ox (if necessary)
  2. Get and install niqlow
  3. Include niqlow on the Ox path. If using OxEdit, have it take input.
  4. Test the installation
  5. Include niqlow in your program

Details of the steps

  1. Get and install Ox and OxEdit.
    2. These instructions presume you are using Ox Console.
    If you have Ox Professional please follow its installation instructions.
    Download from http://www.doornik.com/download/oxmetrics7/Ox_Console/
    It is recommended to install OxEdit as well as Ox Console.
    You can also run Ox from a "command line" or use another programming editor such as WinEdt or Emacs.
  2. Get niqlow
    3. Get the current release or development release from https://github.com/ferrall/niqlow/.
    Extract the zip or gz file. Let PATH/niqlow be the full path to niqlow.
  3. Include niqlow in your Ox path.
    4. For niqlow to work properly, its include folder must be in the include path when you run an Ox program.
    On Windows machines:
    If your program called myprogram.ox relies on niqlow you can run it like this: 
    oxl.exe -i"PATH/niqlow/include" myprogram.ox.
    Again, PATH is where you extracted the niplow folder on your computer.
    One way to do this when using OxEdit is to use Tools->Add/Remove Modules to add -i"PATH/niqlow/include" as an Argument to the Ox run tool.
    If running programs in OxEdit, check on the Takes Input option on the same menu. Otherwise if you run the main examples program nothing will happen because OxEdit will not send your keyboard input to the program.
    See screen shot of including a path in OxEdit and turning on interactive input.
    For Mac Users of Oxedit Only
    Do not put around the path as show above, and do not have any spaces in the folder names in PATH. This is an issue with Oxedit running on Macs (current version as of writing this), not an issue with Ox itself.
    On Unix machines.
    Ox Console on Unix is invoked with the script /OxMetrics6/ox/bin/oxl (or bin64/oxl with the 64-bit version.
    You can modify that script by putting PATH/niqlow/include on the OX6PATH 
    …
    export OX7PATH="\(OXHOME/include;\)OXDEV;$niqlowHOME/include"
    Using CFMPI
    If you are using CFMPI modify the LD_LIBRARY_PATH so that the system can find the shared object files at execution time. You can add a line to the oxl script:
        LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$OXHOME/bin64:PATH/include
  4. Test niqlow by running RosenbrockMain.ox
    5. Inside OxEdit
    Open the file niqlow/examples/RosenbrockMain.ox
    Run the program. Either type Ctrl R or click the first "running man" on the OxEdit menu.
    From a command line prompt 
    PATH/niqlow/examples>  oxl RosenbrockMain
    Check the output
    Either way you run it the output you get should look like the output here: 
     niqlow/examples/output/Rosenbrock.txt
  5. Use niqlow in your Ox program.
    6. Do this by adding this to the top of your .ox or .h file: 
    #import "niqlow"
